# Prompt Engine Architecture

## Prompt layers

The optimizer assembles prompts from layered inputs:

1. System safety and originality rules.
2. Generator rules for tarot, oracle, angel, Lenormand, numerology, philosophy, or hybrid decks.
3. Artistic direction rules.
4. Aesthetic pack adjustments.
5. Reference image analysis.
6. Deck DNA consistency rules.
7. Character consistency rules.
8. Print-quality and model-specific parameters.
9. Negative prompts.
10. User prompt.

## Style injection

Users select simple style labels. The app injects hidden structured rules for medium, lighting, palette, texture, composition, borders, and avoid lists.

## Negative prompts

Negative prompts reduce common generation problems, including visible text, watermarks, malformed hands, distorted faces, celebrity likeness, artifacts, clutter, and low-resolution output.

## Reference image processing

Reference images influence lighting, composition, camera angle, colour harmony, facial proportions, costume, mood, texture, brush style, hierarchy, contrast, depth, and palette. The generated result remains original and should not copy protected work.

## Deck consistency

Deck DNA stores palette, border, lighting, brushwork, motifs, aspect ratio, typography-safe zones, and visual rhythm so new cards match the existing collection.
